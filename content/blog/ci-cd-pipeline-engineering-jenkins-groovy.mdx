# CI/CD Pipeline Engineering with Jenkins Groovy Shared Libraries

Modern platform teams don't write pipelines from scratch for every microservice. At Mareana, I architected a **Groovy-based Jenkins Shared Library (JSL)** framework that standardizes CI/CD across 100+ microservices — reducing pipeline boilerplate by 90% and cutting deployment cycle time from hours to minutes.

Before this framework, every team maintained their own `Jenkinsfile`. Within six months we had 40+ slightly different pipeline definitions — some missing SonarQube gates, some using outdated base images, some with no rollback logic. A single security policy change required updating 40 files. After the Shared Library, it's one change, one PR, one deployment.

---

## The Problem: Pipeline Sprawl

```
Before Shared Libraries:

  service-A/Jenkinsfile   (250 lines, missing SBOM scan)
  service-B/Jenkinsfile   (280 lines, outdated Docker build)
  service-C/Jenkinsfile   (210 lines, no quality gate)
  service-D/Jenkinsfile   (300 lines, hardcoded credentials)
  ...
  service-N/Jenkinsfile   (N variations × N maintenance cost)

After Shared Libraries:

  service-A/Jenkinsfile   (18 lines — calls shared library)
  service-B/Jenkinsfile   (15 lines — calls shared library)
  ...
  jenkins-shared-library/ (ONE versioned, tested codebase)
```

The real cost of pipeline sprawl isn't the initial duplication — it's the ongoing maintenance tax. Every security policy, every tool upgrade, every compliance requirement has to be applied to every pipeline individually.

### How the Library is Loaded

Jenkins loads the shared library at pipeline startup from a Git reference. Libraries are **versioned** — teams pin to a specific tag:

```groovy
// Load the shared library at a specific version tag
@Library('mareana-shared-library@v2.3.1') _
```

This means teams can opt-in to library upgrades on their own schedule — a breaking change in `v3.0.0` won't affect teams still on `v2.3.1`.

## The Complete Pipeline

A service `Jenkinsfile` after adopting the shared library:

```groovy
@Library('mareana-shared-library@v2.3.1') _

pipeline {
  agent {
    kubernetes {
      yaml """
        apiVersion: v1
        kind: Pod
        spec:
          serviceAccountName: jenkins-agent
          containers:
          - name: jnlp
            image: jenkins/inbound-agent:latest
          - name: docker
            image: docker:24-dind
            securityContext:
              privileged: true
          - name: tools
            image: 123456789012.dkr.ecr.us-east-1.amazonaws.com/naik-prod/pipeline-tools:v1.2.0
            command: [cat]
            tty: true
      """
    }
  }

  environment {
    APP_NAME    = 'payment-service'
    DOCKER_REPO = '123456789012.dkr.ecr.us-east-1.amazonaws.com/mareana-prod'
    NAMESPACE   = 'prod-payments'
    HELM_CHART  = './charts/payment-service'
  }

  stages {
    stage('Checkout') {
      steps {
        checkout scm
        // Set build description for traceability
        script {
          currentBuild.description = "Branch: ${env.BRANCH_NAME} | Commit: ${env.GIT_COMMIT[0..7]}"
        }
      }
    }

    stage('Build & Unit Test') {
      steps {
        container('tools') {
          sh 'mvn clean package -DskipTests=false -q'
          // Publish test results
          junit 'target/surefire-reports/*.xml'
          // Archive code coverage
          publishHTML([
            reportDir: 'target/site/jacoco',
            reportFiles: 'index.html',
            reportName: 'Coverage Report'
          ])
        }
      }
    }

    stage('SonarQube Analysis') {
      steps {
        container('tools') {
          runSonarAnalysis(
            projectKey: env.APP_NAME,
            threshold: 80,
            exclusions: '**/generated/**,**/test/**'
          )
        }
      }
    }

    stage('SBOM & Dependency Track') {
      steps {
        container('tools') {
          dependencyTrack(
            projectName: env.APP_NAME,
            version: "${env.BUILD_NUMBER}-${env.GIT_COMMIT[0..7]}"
          )
        }
      }
    }

    stage('Docker Build & Push') {
      when {
        anyOf {
          branch 'main'
          branch 'release/*'
          expression { env.BRANCH_NAME.startsWith('hotfix/') }
        }
      }
      steps {
        container('docker') {
          buildAndPush(
            image: "${env.DOCKER_REPO}/${env.APP_NAME}",
            tag: env.GIT_COMMIT[0..7],
            platforms: 'linux/amd64,linux/arm64'
          )
        }
      }
    }

    stage('Deploy to Staging') {
      when { branch 'main' }
      steps {
        container('tools') {
          deployToKubernetes(
            namespace: 'staging-payments',
            chart: env.HELM_CHART,
            values: 'values-staging.yaml',
            imageTag: env.GIT_COMMIT[0..7]
          )
        }
      }
    }

    stage('Integration Tests') {
      when { branch 'main' }
      steps {
        container('tools') {
          runIntegrationTests(
            namespace: 'staging-payments',
            testSuite: 'payment-integration-tests',
            timeout: 10
          )
        }
      }
    }

    stage('Deploy to Production') {
      when { branch 'main' }
      steps {
        // Manual approval gate for production
        input message: "Deploy ${env.APP_NAME}:${env.GIT_COMMIT[0..7]} to production?",
              ok: 'Deploy',
              submitter: 'platform-team,billing-leads'

        container('tools') {
          deployToKubernetes(
            namespace: env.NAMESPACE,
            chart: env.HELM_CHART,
            values: 'values-prod.yaml',
            imageTag: env.GIT_COMMIT[0..7]
          )
        }
      }
    }
  }

  post {
    always {
      notifySlack(
        status: currentBuild.result ?: 'SUCCESS',
        channel: '#billing-deploys',
        includeChangelog: true
      )
    }
    failure {
      // Auto-rollback if production deploy failed
      script {
        if (env.BRANCH_NAME == 'main') {
          container('tools') {
            sh "helm rollback ${env.APP_NAME} -n ${env.NAMESPACE} --wait"
          }
        }
      }
    }
  }
}
```

## Shared Library Step Implementations

### `buildAndPush.groovy` — Multi-Arch Docker Builds

```groovy
// vars/buildAndPush.groovy

def call(Map config) {
  def image     = config.image     ?: error("image is required")
  def tag       = config.tag       ?: 'latest'
  def platforms = config.platforms ?: 'linux/amd64,linux/arm64'
  def context   = config.context   ?: '.'
  def buildArgs = config.buildArgs ?: [:]

  // Build --build-arg string from map
  def buildArgStr = buildArgs.collect { k, v -> "--build-arg ${k}=${v}" }.join(' ')

  withCredentials([[$class: 'AmazonWebServicesCredentialsBinding', credentialsId: 'aws-ecr', accessKeyVariable: 'AWS_ACCESS_KEY_ID', secretKeyVariable: 'AWS_SECRET_ACCESS_KEY']]) {
    sh """
      # Authenticate to AWS ECR
      aws ecr get-login-password --region us-east-1 | docker login --username AWS --password-stdin 123456789012.dkr.ecr.us-east-1.amazonaws.com

      # Create multi-arch builder (idempotent)
      docker buildx create --use --name multi-arch-builder 2>/dev/null || true
      docker buildx inspect --bootstrap

      # Build and push with layer caching
      docker buildx build \
        --platform ${platforms} \
        --tag ${image}:${tag} \
        --tag ${image}:latest \
        --push \
        --cache-from type=registry,ref=${image}:buildcache \
        --cache-to   type=registry,ref=${image}:buildcache,mode=max \
        ${buildArgStr} \
        ${context}
    """
  }

  // Return the full image reference for downstream steps
  return "${image}:${tag}"
}
```

### `runSonarAnalysis.groovy` — Quality Gate with Timeout

```groovy
// vars/runSonarAnalysis.groovy

def call(Map config) {
  def projectKey = config.projectKey ?: error("projectKey is required")
  def threshold  = config.threshold  ?: 80
  def exclusions = config.exclusions ?: '**/test/**'
  def branch     = env.BRANCH_NAME   ?: 'main'

  withSonarQubeEnv('SonarQube') {
    sh """
      sonar-scanner \
        -Dsonar.projectKey=${projectKey} \
        -Dsonar.projectName=${projectKey} \
        -Dsonar.sources=src \
        -Dsonar.exclusions=${exclusions} \
        -Dsonar.coverage.exclusions=${exclusions} \
        -Dsonar.branch.name=${branch} \
        -Dsonar.qualitygate.wait=true \
        -Dsonar.qualitygate.timeout=120
    """
  }

  // Wait for SonarQube analysis to complete with timeout
  timeout(time: 3, unit: 'MINUTES') {
    def qg = waitForQualityGate()
    if (qg.status != 'OK') {
      // Print failure details before erroring
      echo "Quality Gate FAILED. Status: ${qg.status}"
      echo "Check the SonarQube dashboard for detailed violations."
      error("Pipeline aborted: SonarQube quality gate failed. Coverage must be >= ${threshold}%")
    }
    echo "Quality Gate PASSED ✓"
  }
}
```

### `dependencyTrack.groovy` — SBOM Generation & Supply Chain Security

```groovy
// vars/dependencyTrack.groovy

def call(Map config) {
  def projectName = config.projectName ?: error("projectName is required")
  def version     = config.version     ?: env.BUILD_NUMBER
  def bomFile     = config.bomFile     ?: 'target/bom.xml'

  // Generate CycloneDX SBOM from Maven
  sh """
    mvn org.cyclonedx:cyclonedx-maven-plugin:2.7.9:makeAggregateBom \
      -DschemaVersion=1.4 \
      -DincludeBomSerialNumber=true \
      -DincludeTestScope=false
  """

  // Verify SBOM was generated
  sh "test -f ${bomFile} || (echo 'SBOM generation failed' && exit 1)"

  withCredentials([string(credentialsId: 'dependency-track-api-key', variable: 'DTRACK_API_KEY')]) {
    sh """
      # Upload SBOM to Dependency Track
      HTTP_STATUS=\$(curl -s -o /tmp/dtrack-response.json -w "%{http_code}" \
        -X POST https://dtrack.naik.internal/api/v1/bom \
        -H "X-Api-Key: \${DTRACK_API_KEY}" \
        -H "Content-Type: multipart/form-data" \
        -F "autoCreate=true" \
        -F "projectName=${projectName}" \
        -F "projectVersion=${version}" \
        -F "bom=@${bomFile}")

      echo "Dependency Track response code: \$HTTP_STATUS"
      cat /tmp/dtrack-response.json

      # Fail if upload was unsuccessful
      if [ "\$HTTP_STATUS" != "200" ]; then
        echo "Dependency Track upload failed with status \$HTTP_STATUS"
        exit 1
      fi
    """
  }

  // Wait for analysis and check for critical vulnerabilities
  sleep(time: 30, unit: 'SECONDS')

  withCredentials([string(credentialsId: 'dependency-track-api-key', variable: 'DTRACK_API_KEY')]) {
    def criticalCount = sh(
      script: """
        curl -s -H "X-Api-Key: \${DTRACK_API_KEY}" \
          "https://dtrack.naik.internal/api/v1/metrics/project/${projectName}/current" \
          | python3 -c "import sys,json; d=json.load(sys.stdin); print(d.get('critical',0))"
      """,
      returnStdout: true
    ).trim().toInteger()

    if (criticalCount > 0) {
      unstable("WARNING: ${criticalCount} critical vulnerabilities found in ${projectName}. Review Dependency Track.")
    } else {
      echo "Dependency Track: No critical vulnerabilities found ✓"
    }
  }
}
```

### `deployToKubernetes.groovy` — Helm with Auto-Rollback

```groovy
// vars/deployToKubernetes.groovy

def call(Map config) {
  def namespace  = config.namespace  ?: error("namespace is required")
  def chart      = config.chart      ?: error("chart is required")
  def values     = config.values     ?: 'values.yaml'
  def imageTag   = config.imageTag   ?: error("imageTag is required")
  def release    = config.release    ?: chart.split('/')[-1]
  def timeout    = config.timeout    ?: '5m'

  // Get previous revision for rollback reference
  def prevRevision = sh(
    script: "helm history ${release} -n ${namespace} --max 1 --output json 2>/dev/null | python3 -c \"import sys,json; h=json.load(sys.stdin); print(h[-1]['revision'] if h else 0)\" || echo 0",
    returnStdout: true
  ).trim()

  try {
    sh """
      helm upgrade --install ${release} ${chart} \
        --namespace ${namespace} \
        --values ${values} \
        --set image.tag=${imageTag} \
        --set deployment.timestamp=\$(date +%s) \
        --atomic \
        --timeout ${timeout} \
        --cleanup-on-fail \
        --history-max 10 \
        --wait

      # Verify the rollout
      kubectl rollout status deployment/${release} -n ${namespace} --timeout=3m

      # Quick smoke test — check pods are actually running
      READY=\$(kubectl get pods -n ${namespace} -l app=${release} --field-selector=status.phase=Running --no-headers | wc -l)
      if [ "\$READY" -eq "0" ]; then
        echo "No running pods found after deployment!"
        exit 1
      fi
      echo "Deployment successful: \$READY pods running ✓"
    """
  } catch (Exception e) {
    echo "Deployment FAILED. Attempting rollback to revision ${prevRevision}..."
    sh """
      helm rollback ${release} ${prevRevision} -n ${namespace} --wait --timeout ${timeout} || true
      kubectl rollout status deployment/${release} -n ${namespace} --timeout=3m || true
    """
    error("Deployment failed and was rolled back to revision ${prevRevision}: ${e.message}")
  }
}
```

## Pipeline Security: No Hardcoded Credentials

All credentials use Jenkins' built-in credential store, never environment variables in source code:

```groovy
// PipelineConfig.groovy — centralized credential IDs
// Any change here propagates to all pipelines automatically

class PipelineConfig {
  static final String GCR_CREDENTIALS     = 'gcr-key'
  static final String SONAR_ENV           = 'SonarQube'
  static final String DTRACK_API_KEY      = 'dependency-track-api-key'
  static final String SLACK_CREDENTIALS   = 'slack-webhook'
  static final String HELM_KUBECONFIG     = 'kubeconfig-prod'
  static final String VAULT_ROLE_ID       = 'vault-role-id'

  // Environment-specific namespaces
  static final Map NAMESPACES = [
    staging:    'staging-{app}',
    production: 'prod-{app}',
    qa:         'qa-{app}'
  ]

  static String getNamespace(String env, String app) {
    return NAMESPACES[env]?.replace('{app}', app) ?: "dev-${app}"
  }
}
```

## The Database-Driven Pipeline

At Mareana, we went beyond file-based configuration. Pipeline metadata is stored in a PostgreSQL database, allowing dynamic pipeline configuration without file changes:

```sql
-- Pipeline configuration table
CREATE TABLE pipeline_configs (
  service_name     VARCHAR(100) PRIMARY KEY,
  team             VARCHAR(50),
  deploy_targets   JSONB,          -- ["staging", "production"]
  quality_gate     INTEGER,        -- minimum coverage %
  auto_deploy_prod BOOLEAN,        -- skip manual approval?
  slack_channel    VARCHAR(100),
  helm_chart_path  VARCHAR(200),
  last_updated     TIMESTAMPTZ
);

-- Example row
INSERT INTO pipeline_configs VALUES (
  'payment-service',
  'billing-team',
  '["staging", "production"]',
  85,
  false,
  '#billing-deploys',
  './charts/payment-service',
  NOW()
);
```

```groovy
// K8sClient.groovy — fetch config from DB
def getPipelineConfig(String serviceName) {
  def config = [:]
  withCredentials([usernamePassword(credentialsId: 'pipeline-db-creds',
                                   usernameVariable: 'DB_USER',
                                   passwordVariable: 'DB_PASS')]) {
    def result = sh(
      script: """
        psql postgresql://\$DB_USER:\$DB_PASS@pipeline-db:5432/pipeline_configs \
          -c "SELECT row_to_json(p) FROM pipeline_configs p WHERE service_name='${serviceName}'" \
          -t -A
      """,
      returnStdout: true
    ).trim()
    config = readJSON(text: result)
  }
  return config
}
```

This means a platform admin can change a service's quality gate threshold, Slack channel, or deployment targets from a database UI — no Jenkinsfile change, no PR needed.

## Observability: Pipeline Metrics

We push pipeline metrics to Prometheus via the Pushgateway:

```groovy
// vars/notifySlack.groovy (also pushes metrics)
def call(Map config) {
  def status   = config.status  ?: 'SUCCESS'
  def channel  = config.channel ?: '#platform-deploys'
  def duration = currentBuild.duration / 1000  // seconds

  // Push metrics to Prometheus Pushgateway
  sh """
    cat <<EOF | curl -s --data-binary @- http://prometheus-pushgateway:9091/metrics/job/jenkins_pipeline/instance/${env.APP_NAME}
# TYPE jenkins_build_duration_seconds gauge
jenkins_build_duration_seconds{job="${env.APP_NAME}",branch="${env.BRANCH_NAME}",status="${status}"} ${duration}
# TYPE jenkins_build_total counter
jenkins_build_total{job="${env.APP_NAME}",branch="${env.BRANCH_NAME}",status="${status}"} 1
EOF
  """

  // Send Slack notification
  def emoji   = status == 'SUCCESS' ? '✅' : (status == 'UNSTABLE' ? '⚠️' : '❌')
  def color   = status == 'SUCCESS' ? 'good' : (status == 'UNSTABLE' ? 'warning' : 'danger')
  def message = "${emoji} *${env.APP_NAME}* ${status.toLowerCase()} on `${env.BRANCH_NAME}`\n" +
                "Duration: ${(duration / 60).round(1)}m | " +
                "Build: <${env.BUILD_URL}|#${env.BUILD_NUMBER}>"

  if (config.includeChangelog) {
    def changelog = currentBuild.changeSets.collect { cs ->
      cs.items.collect { "• ${it.msg} (${it.author})" }.join('\n')
    }.join('\n')
    if (changelog) message += "\n\n*Changes:*\n${changelog}"
  }

  slackSend(channel: channel, color: color, message: message)
}
```

## Key Takeaways

1. **Version your library with semver tags** — `@v2.3.1` lets teams upgrade on their own schedule
2. **Fail fast and loudly** — quality gates should block early (after unit tests, not after deploy)
3. **Use `--atomic` on Helm** — always define a rollback strategy before you need it
4. **Centralize credentials** — the credential store ID in `PipelineConfig.groovy` means one place to rotate
5. **Database-driven config beats file-driven config** — operational changes shouldn't require PRs
6. **Instrument everything** — Prometheus metrics on pipeline duration and failure rate expose bottlenecks
